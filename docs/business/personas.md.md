# personas.md

> 목적: TownPet의 **Local(동네) / Market / Care / Lost&Found / Global(Q&A/DB)** 기능을 “누가/언제/왜” 쓰는지 구체화하고, 제품·정책·우선순위 결정을 돕는다.
> 원칙: 각 페르소나는 **트리거(계기) → 불안/제약 → 성공 기준 → JTBD(기능적/정서적/사회적) → 실패 시나리오**까지 포함한다.

---

## 0) 페르소나 공통 정의

### 공통 제약(현실)

* 정보는 많지만 **신뢰/최신성**이 낮다(광고·협찬·바이럴·카더라).
* “반려”는 감정이 크고, 실패 비용이 크다(건강/안전/돈/죄책감).
* 거래/돌봄/실종은 **악용/사기/분쟁**이 기본값이다 → 정책/운영/로그가 제품의 일부.

### 공통 성공 지표(사용자 관점)

* 의사결정 시간 단축(검색→결정)
* 비용/리스크 절감(과잉 진료/헛걸음/사기/사고)
* “불안이 낮아지는 경험”(정서적)
* 반복 행동(저장, 재방문, 알림 반응)

## 1) 페르소나 A: 초보 견주/묘주 (동네 생활 적응자)

### 배경/상황

* 첫 반려동물 입양/분양 후 1~6개월.
* 동네 병원/미용/펫샵/산책 루트를 모른다.
* 커뮤니티 글은 많지만 **내 상황과 맞는지** 판단이 어렵다.

### 트리거(계기)

* 설사/피부/구토 같은 경증 증상, 예방접종, 중성화 상담.
* 산책 중 사고(차량, 다른 개, 농약/오염 구역, 목줄 분쟁).

### 핵심 목표

* “내가 무지해서 아이가 다치거나 돈을 낭비하지 않게.”

### 제약/불안

* 의료는 정보 비대칭(과잉 진료 공포).
* 리뷰가 광고인지 진짜인지 분간 어려움.
* 질문하면 “검색해봐” 같은 반응에 위축.

### JTBD (Functional / Emotional / Social)

* **F**: 병원 선택을 위해 *비용(항목별), 대기, 친절, 설명력*을 비교한다.
* **F**: 안전한 산책 코스를 찾고 *위험 요소(차량, 공사, 자전거, 야간 조명, 유해 식물)*를 회피한다.
* **F**: 사료/간식/알러지/배변 문제를 빠르게 정리하고 “다음 액션”을 얻는다(어떤 증상일 때 병원?).
* **E**: “지금 내 선택이 괜찮다”는 확신이 필요하다(불안 감소).
* **S**: 초보 티가 덜 나게, 비슷한 상황의 사람에게 인정/공감받고 싶다.

### 성공 기준(사용자 정의)

* 병원 방문 전: **예상 비용/대기 범위**를 알고 간다.
* 산책: “불안한 길”을 피하는 루틴이 생긴다.
* Q&A: 같은 질문을 반복 검색하지 않는다(저장/북마크).

### 실패 시나리오(제품이 막아야 함)

* 바이럴 리뷰에 낚여 과잉 진료/비용 폭탄.
* 산책로 정보가 오래돼서 실제와 다름(공사/출입 제한).
* Q&A가 “의사처럼 단정”하며 의료 리스크를 만든다.

### 제품 포인트(요구되는 기능/정책)

* 병원 리뷰는 **템플릿 필수 필드**: 방문일/항목/총액/세부 비용/대기/설명 만족도.
* 산책로는 **시간대 태그**(야간/주말 혼잡) + 안전 태그.
* 초보 전용 온보딩(동네 설정→병원/산책 저장→알림 설정).

## 2) 페르소나 B: 다견/다묘 가구 (반복 비용 최적화자)

### 배경/상황

* 2마리 이상, 또는 노령/만성질환으로 병원 이용 빈도가 높다.
* 사료/모래/패드/간식이 월 고정비로 체감된다.

### 트리거

* 월 지출이 눈에 띄게 늘어남(특히 진료비·검사비).
* 한 번 “호갱” 경험 후, 데이터 기반으로 바꾸고 싶음.

### 핵심 목표

* “반복 비용을 줄이되, 퀄리티는 포기하지 않는다.”

### 제약/불안

* 최저가만 찾다 품질/안전이 떨어질까 걱정.
* 공동구매는 사기/노쇼/분쟁 리스크.
* 중고/대여는 위생·상태·환불 분쟁.

### JTBD

* **F**: 진료 항목별 가격 분포를 보고 합리적 병원을 고른다(“이 동네 평균 대비 비싼가?”).
* **F**: 공동구매로 단가/배송비를 낮추고, 참여/마감/정산을 투명하게 한다.
* **F**: 켄넬/유모차/펜스 같은 고가 장비는 **대여/중고**로 부담을 줄인다(상태 확인).
* **E**: “돈을 아끼는 것”이 아니라 “가족을 잘 돌보는 것”이라는 자기 정당화.
* **S**: ‘정보력 있는 집’으로 인식(커뮤니티 기여).

### 성공 기준

* 월 고정비가 줄어든다(특히 소모품).
* 병원 선택이 감이 아니라 데이터가 된다.
* 거래 분쟁 없이 완료율이 높다.

### 실패 시나리오

* 공동구매 리더 먹튀/미배송.
* 중고 거래 후 하자 분쟁.
* 병원비 데이터가 표본이 적어 오해를 만든다.

### 제품 포인트

* “병원비”는 **항목별 집계/분포(범위)**가 핵심(개별 영수증 수준).
* 공동구매는 *리더 평판/정산 로그/마감 규칙/환불 규칙*이 제품이다.
* 마켓 대여는 **상태 머신 + 체크리스트(인수/반납)**가 필수.

## 3) 페르소나 C: 긴급 돌봄이 필요한 직장인 (시간-리스크 압축자)

### 배경/상황

* 야근, 회식, 출장, 갑작스러운 병원/가족 일정.
* 평소엔 루틴이 있지만, “예외 상황”에서 터진다.

### 트리거

* 내일 갑자기 출장, 오늘 급야근, 예상 못한 일정 변경.
* 산책 공백/약 투약/배변 처리 필요.

### 핵심 목표

* “지금 당장” 신뢰 가능한 사람을 찾고, 사고 없이 끝내기.

### 제약/불안

* 낯선 사람에게 집/반려동물을 맡기는 공포.
* 시간은 없고, 비교는 해야 함.
* 노쇼/연락 두절의 리스크.

### JTBD

* **F**: 30분~2시간 내 가능한 **근거리** 인력을 찾는다.
* **F**: 신뢰도를 빠르게 판단한다(프로필, 인증, 후기, 완료율, 최근 활동).
* **F**: 요청→수락→수행→완료까지 “상태가 한눈에” 보인다.
* **E**: 죄책감(바빠서 못 돌봄)을 줄이고 싶다.
* **S**: 주변에 “민폐” 안 끼치고 해결하고 싶다.

### 성공 기준

* “요청 작성부터 매칭”까지 시간이 짧다.
* 수행 중 안심할 수 있는 업데이트(사진/체크리스트).
* 노쇼/분쟁 없이 완료.

### 실패 시나리오

* 사칭/돌봄 미숙으로 사고.
* 돈 문제로 분쟁(정산/추가 비용).
* 연락 두절/무단 취소.

### 제품 포인트

* 초기에는 결제 없이도 가능하지만, **증빙(사진/체크리스트/타임라인)**은 반드시 필요.
* “긴급”에는 자동 제한이 필요(신규 계정 요청 제한, 인증 없는 공급자 제한).

## 4) 페르소나 D: 도그워커/펫시터 (소규모 공급자, 안정 수입 추구)

### 배경/상황

* 부업/프리랜서/소규모로 시작.
* 플랫폼 없이 인스타/지인 소개 위주 → 수요가 들쭉날쭉.

### 트리거

* 일정이 비거나, 단골 확보가 필요할 때.
* “동네 기반” 리드가 가장 효율적임을 체감.

### 핵심 목표

* 일정 안정화 + 신뢰도 누적으로 단가 방어.

### 제약/불안

* 악성 고객(무리한 요구/환불/리뷰 협박).
* 안전 사고(개 물림, 분실) 책임.
* 노쇼·변경으로 손실.

### JTBD

* **F**: 근거리 요청을 받아 이동 시간을 최소화한다.
* **F**: 스케줄을 관리하고, 반복 고객을 만든다.
* **F**: 후기/완료율/인증으로 “안전한 사람” 신호를 만든다.
* **E**: 노동이 아니라 “전문성”으로 인정받고 싶다.
* **S**: 동네에서 평판을 자산으로 만든다.

### 성공 기준

* 반복 고객 비율 상승.
* 일정 공백 감소.
* 분쟁/신고 비율 낮음.

### 실패 시나리오

* 악성 리뷰로 계정이 망가짐.
* 사고 후 책임 분쟁.
* 공급자끼리 덤핑 경쟁.

### 제품 포인트

* 공급자 프로필은 “예쁜 소개”보다 **리스크 관리 데이터**가 중요:

  * 인증/경력, 완료율, 취소율, 최근 활동, 보험/교육 여부(후반).
* 악성 고객 대응을 위한 **분쟁 프로세스/증빙**이 필요.

## 5) 페르소나 E: 지역 업체(병원/미용/카페/펫샵) (로컬 유입 확대자)

### 배경/상황

* 네이버/인스타 광고 경쟁 심함.
* “반려 친화”를 내세우지만 검증이 어렵다.

### 트리거

* 신규 고객 감소, 리뷰 관리 부담, 재방문 정체.
* 특정 시간대(평일 낮 등) 빈 슬롯을 채우고 싶음.

### 핵심 목표

* 지역 타겟 유입 + 재방문 유도(쿠폰/이벤트).

### 제약/불안

* 광고는 비용이 높고 효율이 낮을 수 있음.
* 부정 리뷰 대응이 어렵다.
* 허위/악성 리뷰 리스크.

### JTBD

* **F**: 반경 기반 노출로 “지금 갈 곳”을 찾는 고객에게 발견된다.
* **F**: 쿠폰/이벤트로 방문을 트리거한다.
* **F**: 문의/예약 흐름을 단순화한다(초기엔 링크, 후반엔 내장).
* **E**: “우리 가게가 신뢰받는 곳”이라는 증거가 필요.
* **S**: 동네에서 좋은 평판을 유지하고 싶다.

### 성공 기준

* 신규 방문 증가(특정 반경/시간대).
* 쿠폰 사용률/재방문율 상승.
* 허위 리뷰/스팸 관리가 가능.

### 실패 시나리오

* 광고 상품이 바이럴/스팸처럼 느껴짐.
* 리뷰 조작 의심.
* 업체가 커뮤니티 신뢰를 망침.

### 제품 포인트

* “업체 노출”은 커뮤니티 신뢰를 해치기 쉬움 → **광고/리뷰/검증 표기 분리**가 필요.
* 업체 계정은 “권한이 더 많다”가 아니라 “투명한 로그/정책 준수”가 더 중요.

## 6) (권장 추가) 페르소나 F: 동네 커뮤니티 리더/중재자 (운영 자원)

> 현재 목록에 없지만, 실제 커뮤니티가 커지면 반드시 등장하는 핵심 페르소나다. “운영 비용”을 줄이는 데 결정적이다.

### 배경/상황

* 동네 단톡/카페에서 이미 도움을 주던 사람.
* 정보 정리/중복 질문 대응/분쟁 중재를 자주 한다.

### 트리거

* 반복 질문/스팸이 증가해 피로도가 높아짐.
* “내가 돕는 방식”을 시스템으로 옮기고 싶음.

### 목표

* 커뮤니티 품질 유지 + 운영 부담 최소화.

### JTBD

* **F**: 신고 큐를 빠르게 처리하고, 기준이 일관되게 적용되길 원한다.
* **F**: 좋은 글(산책로/병원/실종 해결)을 하이라이트/아카이브한다.
* **E/S**: 공정하고 유용한 커뮤니티를 만들었다는 만족.

### 제품 포인트

* 모더레이션 도구(큐/템플릿/자동 블라인드/이의제기) 없으면 성장 구간에서 무너진다.

# JTBD → 기능 맵핑(요약)

## Local

* 병원/장소: **템플릿 리뷰 + 검색/필터 + 저장**
* 산책로: **지도 + 안전 태그 + 시간대 정보**
* 번개/펫친소: **로컬 네트워크 효과(참여/인원/알림)**

## Market

* 판매/대여/나눔: **상태 머신 + 체크리스트 + 후기**
* 분쟁/스팸: **신규 제한 + 신고/자동 블라인드 + 로그**

## Care(마이크로 시팅/산책)

* 긴급 매칭: **반경 + 가용 시간 + 신뢰 지표**
* 수행 증빙: **사진/체크리스트/타임라인**
* (후반) 정산/취소 규칙

## Lost&Found

* 신고: **위치/사진 필수 + 악용 방지**
* 확산: **반경 푸시 + 제보 채널**
* 종료: **해결/감사/검증(허위 방지)**

## Global

* Q&A: **중복 억제 + 답변 품질 + 채택**
* 토픽/DB: **축적/검색성(SEO) + 신뢰 표기**

# 핵심 플로우(현실 버전)

1. **Onboarding**

* 로그인/프로필 → 동네 설정(대표 + 보조) → 관심사(견종/묘종/연령/알러지) → 알림 기본값 설정

2. **Local 탐색 루프**

* 동네 피드/지도 탐색 → 저장/팔로우 → 실제 방문/이용 → 템플릿 리뷰 작성(데이터 축적) → 다음 사용자 의사결정 개선

3. **Market 거래 루프**

* 등록(템플릿) → 문의(채팅 전 단계는 Q&A 형태로 제한 가능) → 상태 변경(예약/판매/취소) → 후기/신고(운영 데이터)

4. **Care 긴급 루프**

* 요청(시간/위치/조건) → 매칭(신뢰/가용성) → 수행(증빙) → 완료/후기(정산은 후반)

5. **Lost&Found 골든타임 루프**

* 신고(필수 필드) → 반경 푸시 → 제보/발견 → 해결 처리 + 허위 방지

6. **Global 지식 루프**

* 질문 작성(템플릿/태그) → 답변 → 채택 → 아카이브/검색 유입
