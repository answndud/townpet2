# ia_navigation.md (개선안) — IA / 내비게이션 설계 (Local vs Global 분리)

> 목적: TownPet의 핵심 전략(Local 리텐션 + Global 검색 유입)을 UI/IA에서 **혼탁 없이 구현**한다.
> 원칙: ① **Local/Global은 정보 구조와 랭킹 로직이 다르므로 섞지 않는다** ② 사용자는 언제든 “지금 어디 모드인지” 인지할 수 있어야 한다 ③ 전환은 자연스럽게 제공하되 기본은 분리.

---

## 0) 결론(짧게)

* 내비게이션은 **Local(생활)** 과 **Global(지식)** 을 ‘탭’만 다르게 두는 게 아니라,
  **피드 구조/작성 템플릿/검색 랭킹/알림 정책**까지 다른 “모드”로 취급한다.
* 홈은 혼합 피드를 만들지 않고, **Local 기본 + Global은 위젯/추천/딥링크**로만 노출한다.

---

## 1) 상위 내비게이션(권장) — “모드 중심”

### 1.1 Bottom/Top Tabs(모바일/웹 공통 개념)

1. **홈** (Local 요약 허브)
2. **동네** (Local 탐색/작성/지도)
3. **지식** (Global 탐색/검색/작성)
4. **마켓** (거래 중심 Local + 상품/정보는 Global 연결)
5. **돌봄** (요청/매칭/수행)
6. **알림**
7. **프로필/설정**

> “마켓/돌봄/실종”은 Local성이 강하지만, 사용자 목적상 독립 탭이 더 빠르다(진입 비용 감소).

### 1.2 상단 ‘모드 배지’(필수 UI)

* 화면 상단에 항상 **[Local] / [Global]** 배지를 표시
* 배지는 클릭 시 해당 모드의 루트로 이동(사용자 위치 감각 유지)

---

## 2) 홈(Home) 정책 — “요약 허브”로 고정

### 2.1 홈 구성(권장 섹션)

* **(A) 내 동네 하이라이트**: 최신/인기/가까운 순 (카드 5~10개)
* **(B) 긴급 배너**: Lost&Found 활성 알림 (우선순위 최상)
* **(C) 바로가기(퀵 액션)**: 리뷰 작성 / 산책로 등록 / 마켓 등록 / 돌봄 요청
* **(D) 내 저장/알림 요약**: 저장한 병원/장소, 키워드 알림, 내 요청 상태
* **(E) Global 위젯(최소 노출)**: 인기 Q&A/추천 토픽 1~2개 + “지식으로 더 보기”

### 2.2 금지 규칙

* 홈을 **무한 혼합 피드**로 만들지 않는다(Local/Global 혼탁 방지)
* 홈에서 Global은 “맛보기/딥링크”만 제공한다(전환은 유도하되 모드는 유지)

---

## 3) Local 영역(동네) — “생활 인프라 모드”

### 3.1 IA(정보 구조)

Local은 “게시판”이 아니라 **생활 카테고리**로 묶는다.

* **병원** (리뷰/비용/대기/항목)
* **동반가능 장소** (카페/식당/공원/숙소 등)
* **산책로** (지도/코스/안전 태그)
* **모임/펫친소** (번개/메이트)
* **실종/유기** (긴급)
* **동네 피드(자유글/잡담)** *(필요 시 최소화)*

> Local의 핵심은 “지금 여기서 의사결정”이므로 카테고리는 행동 중심(병원 가기/산책하기/모임가기)이어야 한다.

### 3.2 Local 기본 UX(규칙)

* 기본 스코프: **내 동네(행정동/생활권)** + 반경 필터
* 기본 정렬: **가까움/최근성/동네 적합도**
* 글쓰기: **템플릿 기반(구조화 필드)** 강제(비교 가능 데이터)
* 지도는 “보조”가 아니라 핵심: 리스트/지도 토글을 1st-class로

### 3.3 Local에서 Global로 연결되는 ‘전환 포인트’

* 병원 리뷰 작성 중 “진료 항목/증상” 입력 → 관련 Q&A 추천(읽기 전용)
* 산책로 태그(야간/안전/대형견) → 관련 가이드/토픽 연결
* 특정 사료/용품 언급 → Global DB 카드 자동 링크

---

## 4) Global 영역(지식) — “검색 유입/지식 자산 모드”

### 4.1 IA(정보 구조)

* **Q&A** (질문/답변/채택/업보트)
* **토픽** (견종/묘종/훈련/건강/행동 등)
* **사료/용품 DB** (성분/알러지 필터 + 리뷰)
* **전문가/가이드** (정책/면책/출처/자격 표기)

### 4.2 Global 핵심 UX(규칙)

* 기본 진입은 **검색 중심**(검색창을 상단 고정)
* 랭킹은 “근접성”이 아니라 **유용성**:

  * 채택/업보트/정확성/최근 업데이트
* 중복 방지:

  * 질문 작성 시 유사 질문 추천(작성 전 차단/유도)
* 의료/훈련 등 고위험 주제:

  * 답변에 “면책/출처/주의” 표기 규칙(정책과 연결)

### 4.3 Global에서 Local로 연결되는 ‘전환 포인트’

* Q&A에서 “지역 추천이 필요”한 경우 → **내 동네 병원/장소 찾기** CTA
* 사료 DB에서 “구매/나눔/공동구매” → **마켓** 딥링크
* 전문가 가이드에서 “근처 산책로” → **동네 산책로** 딥링크

---

## 5) 마켓(거래) IA — “거래 흐름은 Local, 정보는 Global”

### 5.1 마켓 내 탭(권장)

* **내 동네 거래(Local)**: 판매/대여/나눔/공동구매
* **상품 정보(Global 연결)**: 사료/용품 DB 카드, 리콜/주의(Phase)

### 5.2 거래 UX 원칙

* 거래는 “게시판”이 아니라 **상태 머신** 중심(Available/Reserved/Sold 등)
* 대여는 별도 플로우(보증금/기간/인수·반납 체크리스트)

---

## 6) 돌봄 IA — “요청/수행/증빙” 중심

### 6.1 돌봄 내 탭(권장)

* **요청하기** (마이크로 요청 템플릿)
* **받기(매칭)** (반경/시간/조건)
* **내 진행중** (상태/체크리스트/증빙)
* **후기/신뢰** (완료율/취소율/평판)

### 6.2 리스크 제어(IA에 내장)

* 신규 계정은 요청/수락 제한(정책 기반)
* 연락처 노출 대신 인앱 메시지/폼 우선(privacy 문서와 일치)

---

## 7) 알림(Notifications) — Local/Global 분리 운영

### 7.1 알림 카테고리

* **긴급(L&F)**: 최상위 고정 + 반경 기반
* **Local 관심**: 저장한 병원/장소 업데이트, 키워드/반경 알림
* **거래/돌봄 상태**: 예약/수락/취소/상태 변경
* **Global 구독**: 토픽 구독, 내 질문 답변/채택

### 7.2 알림 원칙

* 긴급은 강하게, 나머지는 사용자가 조절 가능(옵트인/옵트아웃)
* 알림에서 딥링크는 **해당 모드 화면**으로 정확히 연결(맥락 유지)

---

## 8) 전역 검색(Global Search) — “토글이 아니라 다른 엔진”

### 8.1 검색 진입점

* 상단 공통 검색바 + 결과 화면에서 **Local/Global 스위치**
* 스위치 변경 시 “필터/정렬/카드 UI”도 같이 바뀌어야 함(사용자 혼란 방지)

### 8.2 랭킹/필터 기본값

* **Local 검색**: 동네/거리/최근성 우선, 지도 결과 지원
* **Global 검색**: 토픽/채택/업보트/최신 업데이트 우선, 중복 질문 억제

---

## 9) URL/라우팅 규칙(개발 친화)

> Next.js App Router 기준으로 “모드가 URL에서 드러나게” 설계한다.

* Local: `/local/...` 또는 현재 구조처럼 `(main)` 그룹 내 `/feed`, `/hospital` 등
* Global: `/global/...` 또는 `/qa`, `/topic`, `/db` 등 “지식 전용 prefix”를 강하게 유지
* 딥링크는 모드를 보존:

  * Local 글에서 Global 카드 클릭 → `/global/...`로 이동하면서 “왜 이동했는지” 상단에 컨텍스트 배지 표시(예: “병원 리뷰에서 연결됨”)

---

## 10) MVP 우선순위(혼탁 방지 체크리스트)

* [ ] 홈은 Local 요약 + Global 위젯(혼합 무한피드 금지)
* [ ] Local/Global 모드 배지를 항상 노출
* [ ] Local/Global 검색은 결과 UI/정렬 로직까지 분리
* [ ] 작성 플로우는 템플릿 기반(특히 Local)
* [ ] 위험 기능(마켓/돌봄/실종)은 독립 탭 + 정책/제한 포함

---

## 11) 리스크/주의(중요)

* “동네 자유글 피드”가 커지면 Local이 잡담 커뮤니티로 흐를 수 있음 →
  리뷰/산책/실종 같은 **구조화 콘텐츠가 항상 우선 노출**되도록 IA에서 강제.
* Global은 SEO가 목적이므로 “토픽/질문”의 URL 구조와 중복 방지가 핵심 →
  질문 작성 흐름이 불편해도(유사 질문 먼저 보여주기) 장기적으로 이득.
