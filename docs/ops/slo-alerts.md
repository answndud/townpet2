# TownPet SLO / Alert Baseline

기준일: 2026-02-19  
목적: 운영 품질 목표를 수치로 고정하고 알람 기준을 명확히 한다.

## 1. SLI 정의

### 1-1. 가용성 (Availability)
- 대상:
  - `GET /api/health`
  - `GET /api/posts`
  - `POST /api/posts` (쓰기 핵심)
- SLI:
  - 성공 응답 비율(2xx/3xx) = 성공 요청 / 전체 요청

### 1-2. 지연시간 (Latency)
- 대상:
  - `GET /api/posts`
  - 게시글 상세 렌더링 (`/posts/[id]`)
- SLI:
  - p95 응답시간(ms)

### 1-3. 정확성/무결성
- 대상:
  - 댓글/반응 후 카운트 정합성
  - 알림 생성/읽음 처리 성공률
- SLI:
  - 정합성 오류 비율
  - 알림 생성 실패율

## 2. SLO 목표 (초기안)
- 월간 가용성:
  - 전체 API 99.5%
  - `GET /api/posts` 99.9%
- 지연시간:
  - `GET /api/posts` p95 < 800ms
  - 게시글 상세 p95 < 1000ms
- 오류율:
  - 5xx 비율 < 1.0% (5분 이동창)
  - 인증/권한 오류(4xx) 급증은 별도 이상징후로 관측

## 3. 에러 버짓
- 월간 에러 버짓:
  - 99.5% 가용성 기준: 0.5% 허용
- 정책:
  - 버짓 50% 소진 시: 신규 기능 배포 속도 제한 + 안정화 우선
  - 버짓 100% 소진 시: hotfix 외 기능 배포 중단

## 4. 알람 규칙 (초기안)

### 4-1. 즉시 알람 (P1)
- 5분 평균 5xx 비율 >= 5%
- `/api/health` 연속 3회 실패
- DB 연결 실패 로그 급증(분당 임계치 초과)

### 4-2. 경고 알람 (P2)
- 15분 평균 5xx 비율 >= 2%
- `GET /api/posts` p95 >= 1200ms (10분 지속)
- 알림 생성 실패율 >= 3% (10분 이동창)

### 4-3. 관찰 알람 (P3)
- OAuth 로그인 실패율 급증
- Rate limit 차단율 비정상 급증

## 5. 대시보드 최소 구성
- 트래픽:
  - RPS, 고유 사용자, 로그인 전환율
- 안정성:
  - 상태코드 비율(2xx/4xx/5xx), 주요 엔드포인트 p95
- 데이터:
  - 게시글/댓글 생성량, 신고 처리량, 알림 생성 성공률
- 인프라:
  - DB 커넥션, 쿼리 지연, 캐시 실패율

## 6. 운영 루프
1. 주 1회 SLO 리뷰
2. 에러 버짓 소진율 점검
3. 상위 에러 코드 Top N 개선 항목 선정
4. `PLAN.md`/`PROGRESS.md`에 반영

## 7. 단계적 고도화
- Stage 1: 로그 기반 수동 집계
- Stage 2: Sentry + 지표 저장소 + 알람 채널 연동
- Stage 3: 서비스별 SLO 분리(피드/작성/알림/인증)
