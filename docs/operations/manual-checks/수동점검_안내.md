# 수동점검 안내

이 폴더는 운영자가 직접 확인해야 하는 점검 절차와 로컬 증적 저장 규칙을 모아 둡니다.

## 저장소에 남기는 문서

- `배포_보안_체크리스트.md`
  - 배포 전후에 반복해서 보는 기준 문서
- `인증_소셜로그인_체크리스트.md`
  - 이메일 로그인, 멀티탭 세션 동기화, 소셜 로그인 연결/복구 점검 절차
- `수동점검_안내.md`
  - 점검 결과를 어디에 어떻게 남길지 정하는 안내 문서

## 저장소에 올리지 않는 파일

- `OAuth_수동점검_기록_YYYY-MM-DD.md`
  - 실행 시각, 증적 링크, 운영 메모가 들어갈 수 있어 `.gitignore`로 제외
- `docs/reports/*.md`
  - 로컬에서 생성되는 측정 결과와 임시 리포트는 커밋 대상에서 제외

## 권장 기록 방식

1. 실행 기준과 판정 기준은 `PROGRESS.md`에 남깁니다.
2. 보안 기준 자체는 `배포_보안_체크리스트.md`에서 관리합니다.
3. 증적 링크와 임시 캡처 경로는 로컬 수동점검 파일에만 남깁니다.
