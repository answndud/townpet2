# 신고/자동 블라인드 정책

목적
- 신고 접수와 자동 블라인드 규칙을 고정한다
- 운영 대응의 기준을 문서화한다

기본 규칙
- 운영 안정 경로 신고 대상: Post
- 신고 API/UI/운영 큐는 게시글(Post)만 지원
- 댓글/사용자 신고는 현재 운영 범위에서 제외하며, 관련 레거시 데이터가 있더라도 신규 접수/처리는 지원하지 않음
- 신고 사유: SPAM/HARASSMENT/INAPPROPRIATE/FAKE/FRAUD/PRIVACY/EMERGENCY/OTHER
- 동일 사용자 중복 신고 금지
- 신고 API는 분당 3회 제한(메모리 레이트리밋)

자동 블라인드
- 신고 3건 고정 규칙은 사용하지 않음
- Post 대상 PENDING 신고는 reporter trust(계정 나이/이메일 인증/활동량/기존 제재 이력)와 신고 속도를 반영해 가중치를 계산
- `고유 신고자 >= 2명`이면서 누적 가중치 `>= 3.0`이면 `Post.status=HIDDEN`으로 자동 숨김
- 저신뢰 신규 계정이 짧은 시간에 몰린 경우에는 즉시 자동 숨김 대신 관리자 큐 우선순위를 `HIGH` 이상으로 올려 수동 검토
- `EMERGENCY` 신고가 포함되면 단건이어도 `CRITICAL` 큐로 승격
- `PRIVACY` 또는 `FRAUD` 신고가 포함되면 최소 `HIGH` 큐로 승격
- 자동 숨김 후보/속도 신호는 관리자 신고 큐에 함께 노출

관리자 처리
- 신고 큐에서 승인/기각 처리
- 관리자 신고 큐와 통계는 Post 대상만 노출
- 기각 시 게시물 상태를 ACTIVE로 복구
- 처리 메모(resolution)를 남길 수 있음
- 일괄 승인 경로도 `applySanction` 옵션을 제공하고, 제재는 사용자별 1회만 부여
- 상태 탭으로 PENDING/RESOLVED/DISMISSED 필터링
- 큐는 `긴급/높음/보통/낮음` 우선순위와 신호 요약(가중치/속도/저신뢰 집중/긴급·개인정보·사기 플래그)을 기준으로 정렬
- 운영 조치 이력은 `ReportAudit`에 기록

비회원 abuse 방어
- 비회원 글/댓글/이미지 업로드는 step-up proof를 요구한다.
- step-up 토큰은 3분 TTL이며 `IP + fingerprint`에 바인딩된다.
- 자동화 UA, fingerprint 부재, 다중 프록시 체인, locale 헤더 부재는 추가 난이도 상승 신호로 취급한다.

데모 환경
- 신고 API 기본 경로는 로그인 사용자(`requireCurrentUser`) 기준으로 처리
- `DEMO_USER_EMAIL`은 로컬/특수 환경 fallback으로만 사용
