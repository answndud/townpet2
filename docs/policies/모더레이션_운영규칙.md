# 02. 모더레이션(신고/제재) 규칙

## 1) 신고 사유(카테고리)
- 허위정보/조작
- 광고/스팸
- 비방/명예훼손
- 사기/금전 요구
- 불법 구인/불법 행위
- 개인정보 노출
- 긴급 위험(자해/타해 위험, 즉시 구조 필요, 현재 진행형 사고)
- 기타(운영자 판단)

## 2) 자동 블라인드 룰(현재 운영 기준)
- 동일 게시물 신고 `>= 3` 고정 규칙은 폐기
- PENDING 신고는 `reporter trust + 계정 연령 + 활동 이력 + 기존 제재 이력 + 10분 내 신고 속도`를 함께 계산
- `고유 신고자 2명 이상`이며 누적 가중치 `>= 3.0`이면 임시 숨김(HIDDEN)
- 저신뢰 계정의 급격한 신고 몰림은 자동 숨김 대신 `HIGH` 우선순위 큐로 승격해 수동 검토
- `긴급 위험(EMERGENCY)` 신고가 1건이라도 포함되면 자동 숨김 여부와 별개로 `CRITICAL` 큐로 즉시 승격
- `개인정보 노출(PRIVACY)` 또는 `사기/거래 위험(FRAUD)` 신고가 포함되면 최소 `HIGH` 큐로 승격
- 현재 운영 신고 대상은 게시글(Post)만 지원
- 사용자 단위 단계적 제재는 운영자 승인 시 `applySanction` 경로로 적용하며, bulk 승인에서도 사용자별 1회만 부여

## 3) 재노출(복구) 기준
- 허위/비방이 아닌 것으로 판단되면 복구
- 템플릿 필드 보완(방문일/항목/금액 등)이 완료되면 복구 가능
- 반복 위반자는 복구 기준을 강화

## 4) 실종/유기 “긴급” 남용 방지
- 긴급 글은 제한 조건(동네 인증/가입 기간/활동 점수) 중 최소 1개를 만족해야 함
- 긴급 글에는 운영자 “신속 검토” 큐를 별도로 운영

## 5) 관리자 액션 로그(Audit)
- 조치 종류: HIDE/UNHIDE/BAN/TEMP_BAN/WARN/EDIT_REQUEST
- 필수 기록: 대상, 사유, 근거(신고 수/로그), 처리자, timestamp
- 신고 큐에는 우선순위(`긴급/높음/보통/낮음`)와 신호 요약(가중치, 속도, 저신뢰 집중 여부, 긴급/개인정보/사기 플래그)을 함께 표시

## 6) 비회원 write abuse 방어
- 비회원 글 작성, 댓글 작성, 이미지 업로드는 모두 step-up 검증을 통과해야 한다.
- step-up은 3분 TTL의 서명 토큰 + SHA-256 proof-of-work 조합으로 구현하며, `IP + fingerprint`에 묶어 재사용을 제한한다.
- 다음 신호가 보이면 난이도를 올린다.
  - fingerprint 없음
  - user-agent 없음 또는 자동화/헤드리스 패턴
  - 과도한 `x-forwarded-for` 프록시 체인
  - `accept-language` 없음
- step-up 실패/위조는 guest violation으로 기록하고, 누적 시 guest ban 기준에 반영한다.
