# TownPet SPEC-Lite

Last updated: 2026-02-26
Purpose: compact engineering spec for day-to-day implementation.

## 1) Product baseline
- TownPet separates `LOCAL` and `GLOBAL` behavior by policy (not just UI filters).
- Global-first flow is enabled: users can read/search/post before neighborhood setup.
- Posting model uses `COMMUNITY` boards and `COMMON` boards (`HOSPITAL`, `LOST_FOUND`, `MARKET`).
- Guest posting is allowed with strict safety policies (rate limits, link/contact restrictions, high-risk category limits).

## 2) Current stack (source of truth: `app/package.json`)
- Next.js App Router (`next@16.x`)
- React (`react@19.x`)
- TypeScript strict
- Prisma + PostgreSQL
- Zod validation
- NextAuth v5 (Credentials + Kakao/Naver)
- Upstash-compatible rate limiting (with fallback)
- Server-validated upload pipeline + `/media/uploads/...` canonical proxy + Vercel Blob/local storage (`@vercel/blob`)
- Vitest + Playwright

## 3) Implementation order (required)
- `Prisma -> Zod -> Service -> Action/Route -> UI -> Tests`

## 4) Architecture boundaries
- API: `app/src/app/api/**`
- Actions: `app/src/server/actions/**`
- Services: `app/src/server/services/**`
- Queries: `app/src/server/queries/**`
- Validation: `app/src/lib/validations/**`
- Schema: `app/prisma/schema.prisma`

## 5) Non-negotiable policy rules
- Keep Local and Global behavior separated in feed/search/alerts/moderation.
- Run policy checks first in high-risk flows (market/care/lost-found/report).
- Enforce rate limits + new-user restrictions on abuse-prone actions.
- Reporting must support `HIDDEN` auto-hide and admin auditability.
- Keep template-first UGC; free text is secondary.

## 6) Security baseline
- Validate all external inputs with Zod.
- Use Prisma parameter binding (avoid raw SQL unless justified).
- Normalize auth failures to 401/403 patterns.
- Avoid PII leakage in logs/errors.

### 7.1 Targeted verification recipes
- `pnpm -C app lint path/to/file.ts path/to/file.test.ts`
- `pnpm -C app typecheck`
- `pnpm -C app test -- path/to/test.spec.ts`
- `pnpm -C app test -- -t "partial test name"`
- `pnpm -C app test:e2e -- path/to/test.spec.ts`
- `pnpm -C app test:e2e -- -g "partial test name"`
- changed-file 기준 lint/test를 먼저 실행하고, schema/shared infra/auth/build script 변경일 때만 전체 Vitest/Playwright로 확장

## 8) Done criteria
- Lint/typecheck/tests pass.
- At least one failure-path test exists for changed behavior.
- `PLAN.md` and `PROGRESS.md` are synchronized.
