# 에러/이슈 처리 기록 규칙

목적
- 개발 중 발생한 에러/이슈의 원인과 해결 과정을 기록한다
- 동일 문제 재발을 방지하고, 운영 정책 개선에 활용한다

기록 규칙
- 파일명: `YYYY-MM-DD_short-topic.md`
- 한 이슈당 한 파일(중복 주제는 새 파일)
- 재현 조건, 원인 분석, 해결 방법, 재발 방지 체크를 포함
- 민감 정보(토큰/비밀키/개인정보)는 절대 기록하지 않는다

기본 템플릿

```
# YYYY-MM-DD short-topic

## 요약
- 증상:
- 영향 범위:
- 심각도:

## 재현 조건
- 단계:
- 환경:

## 원인 분석
- 직접 원인:
- 근본 원인:

## 해결 방법
- 수정 내용:
- 관련 파일:
- 관련 PR/커밋:

## 대응 구분
- 임시 대응(워크어라운드):
- 영구 대응(근본 수정):

## 재발 방지
- 테스트/모니터링 보강:
- 정책/문서 업데이트:
```
