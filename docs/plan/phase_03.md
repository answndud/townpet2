# Phase 3 (Global & Revenue) 개발 순서/프로세스

목적
- Global 지식 기능과 수익화 실험을 단계적으로 확장한다
- 의료/전문 영역 책임 리스크를 정책으로 제어한다

전제
- Phase 2 운영/신뢰 자동화가 안정화됨
- Local/Global 분리 원칙 유지
- PLAN -> SCHEMA -> GENERATE -> REVIEW -> TEST -> DEPLOY 절차 준수

진행 표기 규칙
- [ ] TODO: 착수 전
- [~] DOING: 진행 중
- [x] DONE: 완료
- [!] BLOCKED: 외부 의존/정책 미확정으로 중단

Phase 3 목표
- Global 지식 자산 축적(Q&A, 토픽, DB)
- 검색 유입과 재방문 루프 강화
- 수익화 실험(어필리에이트/멤버십) 기반 마련

세부 순서 (작성/진행 기록용)

1) Q&A (Global)
- [ ] Question/Answer 스키마 및 상태 모델
- [ ] 중복 억제/태그/채택 로직
- [ ] 의료/훈련 표현 정책 가이드(면책/표기)
- [ ] 리스트/상세/작성 플로우

2) 품종/토픽 (Global)
- [ ] 토픽/태그 구조 정의
- [ ] 토픽별 피드/검색/정렬
- [ ] 지식 하이라이트/핀 고정

3) 사료/용품 DB (Global)
- [ ] 제품/성분/알러지 모델 정의
- [ ] 필터/비교 UI 및 검색 최적화
- [ ] 리뷰/추천 로직(정책 포함)

4) 전문가 콘텐츠(정책 선행)
- [ ] 자격/면책/검수 기준 문서화
- [ ] 콘텐츠 표시/검수 플로우
- [ ] 신뢰 배지/표기 규칙

5) 수익화 실험 (정책 선행)
- [ ] 어필리에이트 링크 정책/표기 기준
- [ ] 멤버십 기능 스펙 초안
- [ ] 광고/프로모션 UI 표시 규칙

6) 품질 보강
- [ ] 검색 품질 테스트(중복/정확도)
- [ ] 권한/표기 정책 테스트
- [ ] SEO/메타데이터 기본 세팅

완료 기준(Definition of Done)
- Global 기능이 Local 피드/정책과 분리되어 동작한다
- 의료/전문 영역 책임 리스크가 문서/코드로 통제된다
- 수익화 실험이 정책과 함께 최소 구현된다
